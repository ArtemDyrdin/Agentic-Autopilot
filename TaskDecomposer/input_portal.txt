3. The logic behind the weather widget:‚Äč
‚Äč
The weather widget is integrated into the app to support the AI assistant, which uses current weather
data and user wardrobe information to provide personalized clothing recommendations (e.g., what to
wear based on temperature, precipitation, etc.).‚Äč
‚Äč
The widget is displayed on the app's home page.
It shows weather data exclusively for the current day (today), adapted to the user's location.
‚óŹ‚Äč When the app is launched, the user's geolocation is checked (latitude and longitude).
‚óŹ‚Äč If the location is unique and within the current day (starting from 00:00 UTC) there have been
no requests from users in this location, a query is made to a third-party weather API (e.g.,
OpenWeatherMap or WeatherKit).
‚óŹ‚Äč The obtained data is recorded on the backend and stored there for the period: current day +
3-5 days ahead.
‚óŹ‚Äč If a user from the same location (not unique) opens the app, the app sends a request directly to
the app's backend via the endpoint /weather?lat=...&lon=..., using already stored data, without
additional queries to the external API.
‚óŹ‚Äč When the user interacts with the chat with the AI (assistant), an internal query is made to the
backend using previously obtained weather data (without a new query to the external weather
API if the location hasn't changed).
‚óŹ‚Äč After the query, the weather icon on the widget is updated with new data.
